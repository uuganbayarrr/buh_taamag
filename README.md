# 🤼 Үндэсний бөхийн таамаглагч (Bökh Predictor) — v4

Монгол бөхийн 240,000 гаруй түүхэн барилдааны датаг ашигласан машин сургалтын систем + вэб апп.
Наадмын уур амьсгалтай дизайнтай, **3 модультай**.

## Модулиуд

**1. Хоёр бөхийн барилдаан** — 2 бөх сонгоход ялах магадлал, үзүүлэлтийн харьцуулалт, хоорондын түүх.

**2. Наадам · Тор (Bracket)** — 16 (эсвэл 8/4) бөхийг **гараар оруулна** (эсвэл "Elo-гоор автоматаар
дүүргэх"). Зэрэгцээ нүд (1-2, 3-4 ...) эхний шатанд тулж, үлдсэн 8→4→2→1-ийг загвар тооцно:
- **Monte Carlo** (20,000 симуляц)-аар бөх бүрийн **аварга болох магадлал**
- Хамгийн магадлалт **торны мод** (round бүрийн ялалтын магадлалтай, бөх дээр дарж профайл руу орно).

**3. Бөхийн мэдээлэл** — бөх хайж: **зураг**, Elo/цол/зэрэглэл/бие бялдар, карьерын статистик, бүх
**барилдааны түүх** (огноо, наадам, өрсөлдөгч, дүн). Өрсөлдөгч дээр дарж тухайн бөхийн профайл руу шилжинэ.

**4. 🎯 Таамаглалын тоглоом** — 32 (эсвэл 16) бөхөөс эхлэн **даваа бүрт шалгарах бөхөө сонгоно**;
дараагийн даваанд сонгосон нь л үлдэнэ. Санамсаргүй симуляц "жинхэнэ" үр дүн болж, зөв таасан
бүрт оноо авна (сүүлийн даваанууд илүү өндөр оноотой). Эцэст нь **AI-ийн таамагтай оноогоо
харьцуулж** хэн илүү таасныг харна.

## Цаана юу ажиллаж байна

Барилдаан бүрийг огноогоор эрэмбэлж, **өмнөх** төлвөөс feature тооцоод `HistGradientBoosting`
загвараар P(A давах)-ыг гаргана. Рейтинг: **Elo** (K=40) + **TrueSkill** (μ,σ). Bracket дээр
загвар хосын магадлалын матриц гаргаад олон удаа симуляц хийж аваргыг тодорхойлно.

**Нарийвчлал: ~76.7%, ROC-AUC 0.850** (цаг хугацааны split дээр). Зөвхөн Elo 74.4%, зөвхөн цол 63.7%.

- Апп ажиллуулахдаа орчны хувьсагч зааж өгөх:

## 🔄 Дата шинэчлэгдэхэд загвар автоматаар шинэчлэх

Систем бүхэлдээ таны эх өгөгдлөөс (excel → CSV) тэжээгддэг. **2026 оны шинэ барилдаан нэмэх**:

1. Шинэ мөрүүдийг эх CSV-д ижил баганаар нэмнэ
   (`ulsiin/aimgiin/sumiin buh barildaan.csv`; шинэ бөх бол `... buh.csv`-д).
2. 2 командыг ажиллуулна:
   ```bash
   python3 featurize.py    # Elo, TrueSkill, форм, зэврэлт, барилдааны түүх дахин тооцно (~12 сек)
   python3 fit.py          # K сонголт, загвар дахин сургалт, калибровк, артефакт (~20 сек)
   ```
3. Апп-аа refresh хийхэд бүх зүйл (таамаглал, bracket, профайлын түүх) шинэ өгөгдлөөр шинэчлэгдэнэ.

> `featurize.py` доtorх `UPLOADS` замыг өөрийн CSV хавтас руу зааж өгнө. Excel (.xlsx)-ээс шууд
> уншуулах бол `pd.read_csv`-г `pd.read_excel`-ээр солиход хангалттай.

## Файлууд

```
buh_predictor/
├── featurize.py   # 1-р шат: feature + Elo/TrueSkill + барилдааны түүх
├── fit.py         # 2-р шат: K сонголт, загвар tune, калибровк, артефакт
├── app.py         # FastAPI: predict / bracket / profile / photo / players / topseeds
├── index.html     # 3 модуль, Наадмын дизайн
├── requirements.txt
└── artifacts/     # model.joblib, players.json, h2h.json, history.pkl, metrics.json
```

## API

- `GET  /api/players?q=&active=1` — бөх хайх
- `GET  /api/topseeds?n=16` — Elo-гоор эхний N (bracket-ийг автоматаар дүүргэх)
- `POST /api/predict` — `{"w1","w2"}`
- `POST /api/bracket` — `{"ids":[...]}` (гараар) эсвэл `{"size":16}` (авто)
- `GET  /api/profile/{id}` — мэдээлэл + барилдааны түүх
- `GET  /api/photo/{id}` — зураг (эсвэл орлуулагч)
